# Standardised Spreadsheet Schema

All `sources/*/generate.py` scripts must output a CSV conforming to this schema.
The `builder/` reads these CSVs to generate media and build Anki decks.

---

## Columns

| Column | Description | Example |
|--------|-------------|---------|
| `front_text` | Main body text displayed on the card front | `vivere` |
| `front_labels` | Pipe-separated `label: value` pairs rendered as small highlight chips above the front body text | `infinitive: vivere \| tense: present \| subject: noi` |
| `back_highlight` | Highlighted text on the card back — meaning varies by card type (see below) | `viviamo` |
| `back_text` | Standard body text on the card back — meaning varies by card type (see below) | `we live / we are living` |
| `audio` | Italian text to generate the audio file from | `viviamo` |

---

## `back_highlight` and `back_text` by card type

These two columns are intentionally flexible. Their semantic meaning depends on the type of card being generated:

| Card type | `back_highlight` | `back_text` |
|-----------|-----------------|-------------|
| Verb conjugation | Conjugated form (e.g. `viviamo`) | English meaning (e.g. `we live / we are living`) |
| Noun | Article + noun form (e.g. `la casa`) | English meaning (e.g. `the house`) |

New card types may define their own meaning for these columns; document the convention in the relevant `sources/*/README.md`.

---

## Example rows

```csv
front_text,front_labels,back_highlight,back_text,audio
vivere,"infinitive: vivere | tense: present | subject: noi",viviamo,we live / we are living,viviamo
essere,"infinitive: essere | tense: present | subject: io",sono,I am,sono
```

---

## Rules for source generators

1. The CSV must use UTF-8 encoding.
2. Fields containing commas or pipe characters must be quoted.
3. The `front_labels` field uses ` | ` (space-pipe-space) as the separator between label-value pairs.
4. Each label-value pair uses `label: value` format (space after colon).
